## What is this

A Claude Code Skill. Toss it a "write my daily report" before you log off, and it will:

1. **Dig up the receipts** — scan the local chat histories of 8 AI coding tools (Claude Code, Qoder, Cursor, GitHub Copilot, Aone Copilot, Codex, Cline, OpenCode). Every late-night argument you had with an AI is today's raw material.
2. **Skim the fat** — rewrite everything from your manager's point of view: is the core work moving, what value landed, what's at risk. Renames, refactors and lint fixes — the "proof of effort" — are lovingly discarded.
3. **Draft first, glamour shot later** — you confirm a Markdown draft, *then* it renders a designed HTML report: gradient hero, progress cards, OKR chips, KR progress bars, plus **Copy text / Copy as image** buttons so it lands gracefully in whatever reporting tool your company worships.

Daily reports live in `~/work_journals/daily/`, monthly OKR reports in `~/work_journals/monthly_okr/`.
All the digging happens locally — your secrets never leave home. Only the flattering portrait does.

## Sugar levels (the soul of this skill)

Order like you're at a bubble-tea shop: "sugar-free daily", "monthly report, 30% sugar", "double sugar today". No note means **half sugar**.

| Sweetness | Serves | Tasting notes |
|---|---|---|
| **Sugar-free 0%** | Future you | Straight espresso. The full, honest log — renames, learning sessions and all. Future you remembers nothing, so everything gets written down |
| **30% sugar** | A technical manager | Lightly sweetened. Technical terms and implementation notes survive; connoisseurs will taste the layers |
| **Half sugar 50%** (default) | A regular manager | House recipe. One line of outcome + one line of quantified value per item, jargon translated into human |
| **70% sugar** | A manager who has never seen code | Silky oat-milk smooth. Only "what shipped and what it means for the business" — "P95 340→180ms" becomes "checkout feels faster and steadier" |
| **Double sugar 200%** | A day of, let's say, *strategic recharging* | Dignity mode. **No facts are invented** — small wins simply get a tuxedo: synergize, leverage, operationalize, close the loop, drive alignment, build the flywheel |

Three iron rules, at any sugar level:

1. **Facts and numbers are never fabricated** — sugar is rhetoric, not fraud;
2. The report structure never changes — cards, progress bars and OKR alignment stay on duty;
3. **The sugar level itself never appears in the report** — a double-sugar report your boss can taste is no longer sweet.

## Quick start

Drop the repo into your skills directory:

```bash
git clone https://github.com/unpluggedcoder/sweet-journaler.git ~/.claude/skills/work-journaler
```

Then order away in Claude Code:

```text
write my daily report          # today's report
backfill Tuesday's report      # any date works
monthly report                 # this month's OKR report
sugar-free daily report        # archival honesty, for future you
monthly report, double sugar   # you know what you did (or didn't)
```

The first run asks exactly two questions: the **author name** on your reports, and the **output language** (中文 / English — defaulting to whatever language you triggered the skill in, so "write my daily report" defaults to English). Both are saved to `~/work_journals/config.json` and never asked again; say "this one in Chinese" any day to override just once.
