# O1: 让 NectarSearch 成为业务默认检索入口

## K1: 检索点击率相对 6 月提升 8% — 进度[25% → **60%**]

混合召回灰度进展顺利，点击率增益已在灰度流量上验证，等待扩量后的全量数据。

关键进展：混合检索链路进入 10% 灰度，检索点击率 +6.8%，P95 延迟 210ms → 170ms。

- 完成向量召回索引重建流水线，日更新耗时 4.2h → **1.5h**
- 建成检索质量评测集（1.2k 条标注 query），后续迭代有了统一标尺

## K2: 单位检索成本下降 30% — 进度[40% → **85%**]

两级 embedding 缓存 + 推理动态合批双管齐下，成本目标提前逼近达成。

- 热门 query 向量缓存命中率 41% → **78%**，GPU 推理量日均下降约 1/3
- 推理服务动态合批（dynamic batching）上线，同等流量下 GPU 用卡 12 → **9**

## K3: 检索服务可用性达到 99.95% — 进度[70% → **75%**]

可用性稳步爬升；配置热加载竞态是本月暴露的主要风险，修复已落地待全量。

- 修复排序服务配置热加载竞态，补充回归用例 [[风险]]（全集群发布尚未完成）
- 检索链路接入统一告警，故障平均恢复时长（MTTR）42min → **18min**
